# 투자 연구: 버핏-멍거-돤융핑-리루 4대 거장 종합 분석 프레임워크

$ARGUMENTS 에 대해 체계적인 투자 연구 분석을 수행한다.

## 연구 프레임워크

워런 버핏, 찰리 멍거, 돤융핑(段永平), 리루(李录) 네 명의 투자 거장의 방법론을 기반으로, 다음 7개 모듈을 순서대로 수행한다.

### 사전 단계: AI 연구 편향에 대한 자각 (필수 수행)

연구를 시작하기 전, 해당 기업의 "AI 연구 가능성"을 먼저 평가하여 잠재적인 데이터 편향을 식별한다.

**정보 풍부도 등급**:
| 등급 | 특징 | AI 연구 함정 | 대응 전략 |
|------|------|-----------|---------|
| A등급 (정보 풍부) | 상장된 지 오래되었고, 증권사 커버리지가 많으며 언론 보도가 풍부함 | 시장의 컨센서스가 지나치게 강해 AI 분석 결과가 시장의 기존 평가와 동일해질 가능성이 높음. 초과수익(alpha)을 얻기 어려움 | 반드시 반대 논리를 검증한다. "똑똑한 투자자는 왜 이 회사를 사지 않는가?", "시장이 간과한 위험은 무엇인가?" 를 중점적으로 분석한다. |
| B등급 (정보 보통) | 상장 후 1~3년 정도이며 분석 자료가 제한적이고 일부 데이터는 추정이 필요함 | AI가 부족한 정보를 '합리적인 추론'으로 채워 넣어 겉보기에는 완성도 높은 보고서를 만들지만 실제로는 허위의 확실성을 만들어낼 수 있음 | 모든 추정 데이터에는 반드시 신뢰도(Confidence) 를 표시하고, 근거 있는 추정과 근거 없는 추측을 명확히 구분한다. |
| C등급 (정보 희소) | 최근 상장/비인기 종목/신흥시장, 커버리지가 거의 없음 | AI는 정보 부족으로 인해 지나치게 보수적인 결론을 내리며, "잘 모르겠다 = 좋은 기업이 아니다." 라고 잘못 판단할 가능성이 있음 | 아래의 제1원리(First Principles) 접근법을 사용하여 제한된 정보 속에서도 사업의 본질을 분석한다. |

**C등급 기업을 위한 제1원리 연구법**:
공개 자료가 부족할 때는 "완전해 보이는" 보고서를 억지로 짜맞추려 하지 말고, 다음의 근본적인 질문들에 집중한다.
1. 고객은 누구인가? 왜 돈을 지불하는가? 대체 선택지가 있는가?
2. 재구매는 무엇에 의해 이루어지는가? 습관, 록인(lock-in), 아니면 지속적인 새로운 가치 창출인가?
3. 경쟁사가 100억 원(또는 상당한 자본)을 투자하면 이 사업을 그대로 복제할 수 있는가?
4. 경영진은 지금까지 어떤 중요한 의사결정을 했는가? 그리고 그 결정은 경영진의 어떤 판단력과 가치관을 보여주는가?

**편향 자가 점검 체크리스트** (연구 전 과정에서 경계 유지):
- [ ] 나의 "확신"은 사업의 본질에서 오는가, 아니면 자료의 양에서 오는가?
- [ ] 이 회사에 대한 자료량이 절반으로 줄어든다면, 내 결론이 바뀔까?
- [ ] AI가 도출한 분석이 시장 컨센서스와 지나치게 유사하지 않은가? 만약 그렇다면, 나의 정보 우위는 어디에 있는가?
- [ ] "공개 자료는 적지만 사업의 본질이 매우 우수한" 가능성이 과소평가되고 있지는 않은가?

정보 풍부도 등급 평가 결과를 보고서 서두에 기재하고, 최종 결론에서 "AI 연구 신뢰도"와 "실제 투자 확실성"의 차이를 명시한다.

### 1단계: 데이터 수집

> **데이터 소스 규범**: `skills/financial-data.md` 참조. 모든 재무 데이터는 반드시 서로 독립된 두 개의 출처에서 가져와야 하며, 오차가 1%를 초과하면 표시해야 한다.
> - 미국 주식: macrotrends(주) + stockanalysis(부)
> - 홍콩 주식: aastocks(주) + macrotrends ADR(부)
> - 중국 A주: 둥팡차이푸(東方財富, 주) + 쥐차오쯔쉰(巨潮資訊, 부)
> - 대한민국 주식: 에프앤가이드(주) + OpenDART(부)

Task 도구를 사용하여 백그라운드 에이전트를 실행하고, 웹에서 다음 데이터를 수집한다.

1. 매출 구조: 최근 회계연도 및 최근 4개 분기의 부문별 매출, 성장률, 매출총이익률
2. 재무 지표: 최근 5년 매출, 순이익, 매출총이익률, 영업이익률, 잉여현금흐름, 현금 보유고
3. 경쟁 구도: 시장 점유율, 주요 경쟁사 비교
4. 비즈니스 모델과 해자(垓子): 핵심 경쟁 우위의 원천
5. 기술 역량: 핵심 기술 스택, R&D 투자
6. 경영진: 창업자/CEO 이력, 지분 보유 비율, 핵심 의사결정 기록
7. 산업 전망: TAM(전체 목표 시장 규모), 성장 예측
8. 리스크 요인: 지정학, 규제, 공급망 등
9. 현재 밸류에이션: 시가총액, PER, PSR, PEG, EV/매출
10. 매수·매도 양측의 핵심 논거

#### 데이터 교차 검증 (필수 수행, 금융 정밀성 도구 사용)

데이터 수집이 끝나면, **반드시 `tools/financial_rigor.py`를 호출하여 핵심 데이터를 프로그램적으로 검증**하여 LLM의 암산 오류를 방지해야 한다.

**반드시 검증해야 할 데이터 항목**:
- 총 발행주식수 (거래소, Yahoo Finance, StockAnalysis 등 최소 2개 출처에서 확인)
- 현재 주가 및 시가총액 (**주가×총 발행주식수를 직접 계산하여 보고서상의 시가총액과 대조하고, 단위 오류를 방지**)
- 최근 회계연도 매출 및 순이익 (기업 연차보고서 + 최소 1개의 제3자 출처에서 확인)
- 현금 보유고 및 순현금 (현금+단기투자-총부채, 산정 기준의 차이에 유의)
- 경영진 지분 보유 비율 (경제적 지분과 의결권을 구분하고, 차등의결권(AB주) 구조에 유의)

**필수 검증 절차 (Bash를 통한 도구 호출)**:

1단계 — 시가총액 검산 (부동소수점이 아닌 정확한 십진수 사용):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap \
  --price {주가} --shares {총 발행주식수} --reported {보고된 시가총액} --currency {통화}
```

2단계 — 핵심 데이터 다중 출처 교차 검증:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate \
  --field {항목명} --values '{"출처1": 값, "출처2": 값}' --unit {단위}
```
매출, 순이익, 현금 보유고에 대해 각각 실행한다.

3단계 — 밸류에이션 지표 정밀 검산 (PER/PBR/ROE/FCF 수익률 등):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation \
  --price {주가} --eps {주당순이익} --bvps {주당순자산} --fcf-per-share {주당 FCF} --dividend {주당배당금}
```

**검증 규칙**:
1. 모든 핵심 데이터 항목은 최소 2개의 독립된 출처를 갖는다
2. 출처 간 차이가 발견될 경우, 기업 연차보고서/거래소 데이터를 우선 채택하고 차이의 원인을 기재한다
3. **계산이 포함된 모든 데이터는 반드시 도구로 검산해야 하며, LLM의 암산을 금지한다**
4. 도구 출력 결과는 보고서 부록 "핵심 데이터 교차 검증 기록"에 직접 삽입한다
5. 도구가 ❌(편차 과다)를 보고할 경우, 원인을 규명한 후에만 분석을 계속할 수 있다

**흔한 오류 방지**:
- 시가총액 단위: 홍콩달러 억 vs 위안화 억 vs 미국달러 억 — 0을 하나 빠뜨리거나 더하기 쉬움
- FCF 산정 기준: 출처마다 자본적지출(CapEx)의 정의가 다를 수 있음(임대, 인수 포함 여부 등)
- 부채 산정 기준: 운용리스 부채 포함 여부
- 지분 보유 비율: 차등의결권(AB주) 기업의 경제적 지분 ≠ 의결권

### 2단계: 사업 본질 분석 — 돤융핑의 "옳은 사업(對的生意)"

분석 요점:
- 이 사업의 본질을 한 문장으로 정의한다
- 매출 구조 분해 (도표)
- 5년간 수익성 추세 (도표)
- 비즈니스 모델 캔버스: 일회성 판매 vs 구독/재구매? 하드웨어 vs 소프트웨어 vs 플랫폼?
- 생태계 점착성/고객 록인 강도
- 매출총이익률 수준 및 동종업계 비교, 높거나 낮은 이유 설명
- 영업 레버리지 분석
- **돤융핑식 질문**: 이 사업의 좋은 점은 무엇인가? 한 문장으로만 설명한다면 무엇인가?

### 3단계: 해자 평가 — 버핏의 "경제적 해자"

다섯 가지 유형의 해자를 하나씩 검증한다.

| 해자 유형 | 검증 방법 |
|-----------|---------|
| 브랜드/가격결정력 | 판매량 감소 없이 가격을 인상할 수 있는가? |
| 전환 비용 | 고객이 경쟁사 제품으로 이전하는 데 드는 비용은 얼마나 높은가? |
| 네트워크 효과 | 사용자가 많아질수록 제품이 더 좋아지는가? |
| 규모의 경제 | 규모가 가져다주는 원가 우위는 얼마나 큰가? |
| 기술/특허 장벽 | 기술적으로 몇 년이나 앞서 있는가? 복제될 수 있는가? |

해자의 추세를 분석한다: 지난 5년간 넓어졌는가, 좁아졌는가? 향후 5년은 어떻게 예상되는가?

**버핏식 질문**: 10년 후에도 이 해자는 존재할까? 무엇이 이를 무너뜨릴 수 있는가?

### 4단계: 역발상 사고와 리스크 목록 — 멍거의 "거꾸로 생각하기(反過來想)"

- "이 회사가 실패할 수 있는 모든 경로"를 나열한다 (표: 경로/확률/영향 정도)
- 역사적 유추: 과거 비슷한 위치에 있었던 회사를 찾아보고, 결말이 어땠는지 확인한다
- 학제간 분석: 네트워크 효과 이론, 기술 수용 곡선, 경쟁 게임 이론 등의 모델로 교차 검증한다
- 편향 자가 점검: 서사 편향, 앵커링 효과, 생존자 편향
- 매도(공매도) 측의 핵심 논거를 수집한다

**멍거식 질문**: 내가 가장 오류를 범하기 쉬운 지점은 어디인가? 똑똑한 사람들은 왜 이 회사를 사지 않거나 공매도하는가?

### 5단계: 경영진 평가 — 돤융핑의 "옳은 사람(對的人)" + 버핏의 "경영진의 진실성"

- CEO/창업자의 핵심 의사결정 복기 (표: 시기/결정/결과/평가)
- 자본 배분 능력: R&D 수익률, 인수합병 성공률, 자사주 매입 시점
- 주주 이익 일치성: 경영진 지분 보유, 보수 구조, 지분 매각 기록
- 조직 역량: 팀 안정성, 핵심 인재 리스크
- 기업 문화 특징

**돤융핑식 질문**: CEO가 은퇴한다면, 이 회사는 여전히 경쟁력을 유지할 수 있는가?

### 6단계: 산업 및 문명적 추세 — 리루의 "문명 진화 프레임워크"

- 해당 산업이 "문명적 차원의 패러다임 전환" 단계에 있는지 판단한다
- 역사적 기술 혁명과의 유추 (증기기관/전기/인터넷/AI)
- TAM 성장 곡선과 상한선 분석
- 산업 가치사슬에서 이 회사의 위치
- 기술 경로 리스크
- 고객/공급업체 집중도 분석

**리루식 질문**: 20년 후 돌아봤을 때, 이 회사는 "이 시대의 스탠더드 오일"인가, 아니면 "반짝했다 사라진 3Com"인가?

### 7단계: 밸류에이션과 안전마진 — 버핏의 "내재가치" + 돤융핑의 "옳은 가격(對的價格)"

- 현재 시장 가격 (핵심 밸류에이션 지표 표) — **반드시 도구로 검산해야 함**
- 역산 DCF: 현재 주가는 어떤 성장률 기대를 내포하고 있는가?
- 3가지 시나리오 밸류에이션 — **반드시 도구로 정밀 계산해야 하며, 암산을 금지**:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario \
  --price {주가} --eps {주당순이익} --shares {총 발행주식수(억)} \
  --growth {낙관 성장률} {중립 성장률} {비관 성장률} \
  --pe {낙관 PER} {중립 PER} {비관 PER} --years 3 --currency {통화}
```
- 자체 과거 밸류에이션과의 비교
- 동종업계 밸류에이션과의 비교

**돤융핑식 질문**: 만약 주식시장이 내일부터 5년간 문을 닫는다면, 이 가격에 계속 보유하고 싶은가?

## 출력 요구사항

1. 모든 분석은 데이터에 근거해야 하며, 출처를 첨부해야 한다
2. 핵심 데이터는 마크다운 표로 제시한다
3. 각 모듈의 끝에는 해당 거장의 "질문"이 반드시 있어야 한다
4. 최종적으로 완성된 보고서를 `~/[회사명]투자연구보고서.md`에 작성한다
5. 결론은 명확해야 하며, 매수/관망/회피 중 하나의 권고를 회피하지 않는다
6. 밸류에이션 부분에는 반드시 구체적인 가격 범위를 제시해야 한다
7. **보고서 서두**에는 "정보 풍부도 등급"(A/B/C)과 "AI 연구 한계 고지"를 반드시 포함해야 한다
8. **보고서 말미**에는 "AI 분석 신뢰도"와 "투자 확실성"을 반드시 구분해야 한다 — 전자는 자료의 양에 좌우되고, 후자는 사업의 본질에 좌우된다. 이 보고서의 어떤 결론이 충분한 데이터에 근거한 것이고, 어떤 결론이 제한된 정보에 기반한 추론인지 독자에게 명확히 알려야 한다
9. 만약 회사가 C등급(정보 희소)에 해당한다면, 보고서 말미에 "직접 검증이 필요한 질문 목록"을 반드시 나열해야 한다 — 독자가 현장 조사, 제품 체험, 공급망 인터뷰 등의 방식을 통해 AI의 사각지대를 보완하도록 권고한다

## 데이터 표본 검사 (배포 승인 절차)

보고서 작성이 완료된 후, **반드시** 데이터 표본 검사를 수행해야 하며, 통과해야만 발행할 수 있다.

**1단계 — 표본 검사 목록 추출 (15% 무작위 표본추출)**:
```bash
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <보고서 파일 경로>
```
JSON 템플릿을 출력한다. 각 항목에는 `fetched_value`(입력 대기)가 포함된다.

**2단계 — 데이터 실사 검증**:
목록의 각 데이터 항목에 대해, `skills/financial-data.md` 규범에 따라 신뢰할 수 있는 출처에서 데이터를 취득한다
(미국 주식: macrotrends+stockanalysis; 홍콩 주식: aastocks+macrotrends; 중국 A주: 둥팡차이푸+쥐차오쯔쉰),
`fetched_value` / `fetched_source` / `fetched_value2` / `fetched_source2`에 기입한다.

**3단계 — 판정 결과 출력**:
```bash
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<작성 완료된 JSON>' \
  --report <보고서 파일명>
```

- **【승인】**: 모든 표본 검사 항목의 편차 ≤ 1% → 보고서 발행 가능
- **【반려】**: 어느 한 항목이라도 편차 > 1% → 해당 데이터를 수정한 후 재검사, 승인될 때까지 반복

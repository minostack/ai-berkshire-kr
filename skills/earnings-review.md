# 실적 정밀 분석: 1차 자료 심층 해독

$ARGUMENTS 에 대한 실적 정밀 분석을 수행합니다.

**지원 입력 형식**: `기업명 분기`, 예: `삼성전자 2025Q4`, `카카오 2025년보`, `현대차 최신` (기본값: 가장 최근 분기)

> "나는 셀사이드 리포트를 보지 않는다. 오직 원본 재무제표만 읽는다." — 리루
>
> "나는 매일 500페이지를 읽는다. 지식은 그렇게 쌓인다. 복리처럼." — 버핏

## 설계 철학

대부분의 AI 투자 리서치 도구는 2차 정보(뉴스, 리포트 요약, 데이터 사이트)에 의존합니다. 하지만 버핏과 리루의 핵심 역량은 **1차 자료를 직접 읽는 것**입니다 — 연간 보고서, 분기 보고서, 실적 발표 컨퍼런스 콜.

2차 정보의 문제점:
- 걸러진 정보 — 애널리스트가 자신의 관점에 유리한 데이터를 선택적으로 제시
- 시간 지연 — 남들이 소화하고 나면 알파는 이미 사라진 뒤
- 맥락 부재 — "매출 15% 성장"은 경영진이 성장 품질에 대해 어떻게 말했는지 맥락이 빠져 있음

이 스킬은 1차 자료를 직접 해독하여, 버핏과 리루가 실제로 주목하는 내용에 집중합니다.

## 실행 흐름

### 전제 단계: 자료 확보 가능성 등급

| 등급 | 특징 | 영향 |
|------|------|------|
| A등급 | 원문 전체 확보 (10-K / 연간 보고서 / 컨퍼런스 콜 기록) | 전체 단계 정상 실행 |
| B등급 | 원문 일부 또는 제3자 요약만 확보 | "비원본 출처" 표기, 주석 분석 비중 축소 |
| C등급 | 뉴스 보도 및 데이터 사이트 요약만 있음 | 핵심 재무 데이터 변화에 집중, 주석 발굴 생략, "1차 자료 부족" 표기 |

### 1단계: 1차 자료 확보

Task 도구를 사용하여 여러 백그라운드 에이전트를 **병렬**로 시작하여 아래 원본 자료를 확보합니다:

1. **재무제표 원문**: 기업 IR 페이지, SEC EDGAR(미국주 10-K/10-Q), 홍콩거래소 공시(홍콩주), 전자공시시스템 DART(한국주)에서 확보
2. **실적 컨퍼런스 콜 기록/녹음**: Seeking Alpha, 기업 IR 페이지, 네이버 금융 등에서 확보
3. **경영진 주주 서한** (연간 보고서의 경우): 전문 완독
4. **투자자 설명회 / 애널리스트 데이 자료** (최근에 있었다면)

원문 확보가 불가능한 경우, `skills/financial-data.md` 기준의 표준 데이터 출처를 사용합니다 (미국주: macrotrends+stockanalysis; 홍콩주: aastocks+macrotrends; 한국주: 전자공시시스템+네이버금융). 단 반드시 "원본 재무제표가 아닌 제3자 요약"임을 표기하고, 핵심 데이터의 두 출처 간 오차 >1%이면 표시합니다.

### 2단계: 핵심 재무 데이터 추출 및 검증

#### 2.1 매출 및 손익계산서

| 지표 | 당기 | 전기 | YoY 변화 | 경영진 가이던스 | 달성 여부 |
|------|------|------|---------|--------------|---------|

반드시 포함:
- 총매출 및 사업부문별 / 지역별 매출 분해
- 매출총이익, 매출총이익률 변화
- 영업이익, 영업이익률 변화 (GAAP vs Non-GAAP 구분)
- 순이익 (비경상 손익 영향 주의)
- EPS (기본 vs 희석)

#### 2.2 현금흐름표 (버핏이 가장 중시)

| 지표 | 당기 | 전기 | 변화 | 주목 포인트 |
|------|------|------|------|-----------|

반드시 포함:
- 영업현금흐름 vs 순이익 비율 (>100% 양호, <80% 경계)
- 자본적 지출 및 구성 (유지성 vs 확장성)
- 잉여현금흐름 = 영업현금흐름 - 자본적 지출
- 자사주 매입 금액, 배당금 지급액
- 기말 현금 및 현금성 자산

#### 2.3 재무상태표 건전성

반드시 포함:
- 현금+단기 투자자산 vs 이자부 부채
- 순현금/순부채 변화 추이
- 매출채권 회전일수 변화 (신용 조건 완화로 매출 밀어내기 여부?)
- 재고자산 회전일수 변화 (적체 여부?)
- 영업권 및 무형자산 비중 (손상 리스크 여부?)

**데이터 검증**: `tools/financial_rigor.py`로 핵심 데이터 검증:

```bash
# 매출 및 순이익 교차 검증 (최소 2개 출처)
python3 tools/financial_rigor.py cross-validate \
  --metric "revenue" --values 108.3e9 107.9e9 --sources "회사 재무제표" "네이버 금융"

# 시가총액 검증
python3 tools/financial_rigor.py verify-market-cap \
  --price 101 --shares 1.488e9 --reported 1.44e11 --currency KRW

# 밸류에이션 지표 검증
python3 tools/financial_rigor.py verify-valuation \
  --price 101 --eps 9.6 --bvps 26.5 --fcf-per-share 10.2
```

### 3단계: 경영진 논의 정밀 독해 (MD&A)

버핏과 리루가 가장 많은 시간을 투자하는 부분입니다. 숫자가 아니라 **경영진이 어떻게 말하는지**를 봅니다.

#### 3.1 경영진 어조 분석

경영진 논의/컨퍼런스 콜 발언을 단락별로 읽으면서 아래 신호를 표시합니다:

| 신호 유형 | 구체적 표현 | 예시 |
|---------|-----------|------|
| 🟢 **솔직한 신호** | 문제를 먼저 인정하고 구체적 원인 제시 | "이번 분기 이익률 하락은 X 영역 투자가 예상을 초과했기 때문입니다" |
| 🟢 **명확한 신호** | 전략이 구체적이고 정량적 목표 제시 | "향후 12개월 내 X 사업의 시장 점유율을 15%에서 20%로 높일 계획입니다" |
| 🔴 **모호한 신호** | "우리는 믿는다", "장기적으로" 등 실질적 내용 없는 표현 남발 | "미래에 대해 충분히 자신합니다" |
| 🔴 **회피 신호** | 직접적인 질문을 피하고 다른 주제로 전환 | 이익률 질문에 매출 성장 얘기로 전환 |
| 🔴 **외부 귀인 신호** | 문제를 거시환경/업계/경쟁사 탓으로만 돌림 | "거시환경의 영향으로..." |

#### 3.2 약속 추적

전기 재무제표/컨퍼런스 콜에서 경영진의 구체적 약속을 추출하여 당기 실제 결과와 비교합니다:

| 전기 약속 | 당기 이행 결과 | 평가 |
|---------|------------|------|
| "하반기에 이익률이 X%로 회복될 것" | 실제 Y% | ✅달성 / ❌미달 / ⚠️일부 달성 |

**단용핑**: "경영진을 믿을 수 있는지 판단하는 가장 간단한 방법은, 이전에 한 말을 실제로 지켰는지 보는 것이다."

#### 3.3 핵심 질문 식별

컨퍼런스 콜 Q&A에서 애널리스트의 가장 날카로운 질문과 경영진의 답변 품질을 추출합니다:

| 애널리스트 질문 | 경영진 답변 | 답변 품질(1~5) | 회피 여부 |
|-------------|-----------|:----------:|:------:|

### 4단계: 주석 및 숨겨진 정보 발굴

재무제표 주석에는 경영진이 쉽게 보여주고 싶지 않은 정보가 숨어 있습니다:

#### 4.1 필수 확인 주석 항목

- [ ] **특수관계인 거래**: 대주주/특수관계인과의 거래 조건이 공정한가?
- [ ] **주식보상**: 스톡옵션/RSU의 희석 효과는 얼마나 되는가? 행사가는?
- [ ] **우발부채**: 소송, 보증, 확약 등 부외 리스크
- [ ] **회계 정책 변경**: 수익 인식 방식, 감가상각 연한 등의 변경 여부
- [ ] **부문 정보**: 사업 부문별 이익률 차이, "좋은 사업이 나쁜 사업을 보전"하는지 여부
- [ ] **고객/공급업체 집중도**: 상위 5개 고객/공급업체 비중

#### 4.2 이상 신호 탐지

- [ ] 매출채권 증가율 > 매출 증가율 (채널 밀어내기 가능성)
- [ ] 재고자산 증가율 > 매출 증가율 (적체 가능성)
- [ ] 영업현금흐름 < 순이익이고 격차 확대 (이익 품질 의심)
- [ ] 자본화 지출 급증 (이익 미화 가능성)
- [ ] 비경상 이익 비중 급상승

### 5단계: 과거 데이터와의 비교

#### 5.1 추세 분석

당기 핵심 지표를 최소 4개 분기(또는 3년 연간) 시계열에 배치합니다:

| 지표 | Q-4 | Q-3 | Q-2 | Q-1 | 당기 | 추세 판단 |
|------|-----|-----|-----|-----|------|---------|

중점 확인:
- 이익률이 개선 중인가, 악화 중인가?
- 매출 성장률이 가속 중인가, 둔화 중인가?
- 현금흐름 품질이 향상 중인가, 저하 중인가?
- 자본적 지출 강도가 증가 중인가, 감소 중인가?

#### 5.2 경영진 가이던스와의 비교

| 지표 | 경영진 이전 가이던스 | 실제 결과 | 편차 | 해석 |
|------|-----------------|---------|------|------|

### 6단계: 정밀 분석 리포트 출력

#### 리포트 구조

```
1. 핵심 데이터 요약 (한 페이지 표)
2. 이번 분기 가장 중요한 변화 3가지 (500자 이내)
3. 경영진 어조 및 약속 추적
4. 주석에서 발굴한 숨겨진 정보
5. 핵심 질문 (컨퍼런스 콜 Q&A 정선)
6. 투자 논거와의 관계 (보유 포지션이 있는 경우)
7. 결론: 이번 실적이 무엇을 바꿨는가?
```

#### 결론은 반드시 아래 질문에 명확히 답해야 합니다

1. **이번 실적은 예상 대비 어떠한가? (시장 기대 상회 / 부합 / 하회)** ("대체로 부합" 후 양면 나열은 불가)
2. **투자 논거에 미치는 영향**: 강화 / 영향 없음 / 약화 / 붕괴
3. **다음으로 주목해야 할 촉매는 무엇인가?**
4. **현재 보유 중이라면 추가 매수 / 유지 / 매도?**

### 7단계: 리포트 저장

리포트를 `reports/{기업명}-earnings-{기간}.md`로 저장합니다. 예: `reports/삼성전자-earnings-2025Q4.md`

### 8단계: 데이터 검수 (출고 전 검증)

리포트 저장 후 데이터 검수를 실행하고, 통과 후 발행합니다:

```bash
# Step 1 — 검수 목록 추출
python3 tools/report_audit.py extract \
  --report reports/{기업명}-earnings-{기간}.md

# Step 2 — 목록 각 항목을 신뢰할 수 있는 출처에서 데이터 확인 (skills/financial-data.md 참조)

# Step 3 — 출고 승인 / 반려 판정 출력
python3 tools/report_audit.py verdict \
  --results '<완성된 JSON>' \
  --report {리포트 파일명}
```

**【출고 승인】** 전체 통과 → 발행; **【반려】** 미통과 항목 있음 → 수정 후 재검수.

## 핵심 원칙

- **원문을 읽고, 요약을 읽지 않는다**: 가능한 모든 방법으로 1차 자료를 확보합니다
- **변화를 보고, 절댓값을 보지 않는다**: 추세가 숫자 그 자체보다 중요합니다
- **어조를 듣고, 내용만 듣지 않는다**: 경영진이 어떻게 말하는가가 무엇을 말하는가만큼 중요합니다
- **주석을 확인하고, 본문만 보지 않는다**: 디테일 속에 진실이 있습니다
- **결론을 내리고, 요약에 그치지 않는다**: 정밀 분석의 목적은 판단을 형성하는 것이지 재무제표를 다시 서술하는 것이 아닙니다
